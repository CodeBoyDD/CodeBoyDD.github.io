CentOS 7 下源码安装 Nginx

info

CentOS 7 下源码安装 Nginx 并使用 systemctl 管理 nginx 服务

下载 Nginx 源码包​

wget http://nginx.org/download/nginx-1.18.0.tar.gz

解压源码包并进入安装目录

tar -zxvf nginx-1.18.0.tar.gz && cd nginx-1.18.0

创建 nginx 用户及用户组​

sudo groupadd nginx && sudo useradd -r -g nginx -s /sbin/nologin nginx
info

构建参数文档: http://nginx.org/en/docs/configure.html

CSDN汉化: https://blog.csdn.net/guojing1173132123/article/details/83623483

配置 Nginx 安装选项​

./configure \
--prefix=/usr/local/nginx \
--sbin-path=/usr/local/nginx/sbin/nginx \
--conf-path=/usr/local/nginx/conf/nginx.conf \
--error-log-path=/usr/local/nginx/logs/error.log \
--pid-path=/usr/local/nginx/logs/nginx.pid \
--lock-path=/usr/local/nginx/logs/nginx.lock \
--http-log-path=/usr/local/nginx/logs/access.log \
--user=nginx \
--group=nginx \
--with-http_ssl_module
tip
参数参数说明
--prefix指定 nginx 安装目录; 默认: /usr/local/nginx
--sbin-path指定执行程序文件存放位置; 默认: prefix/sbin/nginx
--conf-path指定配置文件存放位置; 默认: prefix/conf/nginx.conf
--error-log-path指定错误日志存放位置; 默认: prefix/logs/error.conf
--pid-path指定 pid文件存放位置; 默认: prefix/logs/nginx.pid
--lock-path指定 lock文件存放位置; 默认: prefix/logs/nginx.lock
--http-log-path指定access.log路径; 默认: prefix/logs/access.log
--user指定用户; 默认: nobody
--group指定用户组; 默认: nobody
--with-http_ssl_module支持https协议,需要openssl

编译安装​

sudo make -j $(nproc) && sudo make install

创建软连接​

sudo ln -s /usr/local/nginx/sbin/nginx /usr/bin/

设置nginx应用目录权限​

sudo chown -R nginx:nginx /usr/local/nginx && sudo chmod -R 755 /usr/local/nginx

使用systemctl管理nginx服务​

sudo vim /usr/lib/systemd/system/nginx.service

编辑文件内容

/usr/lib/systemd/system/nginx.service
[Unit]Description=nginxAfter=network.target[Service]Type=forkingExecStart=/usr/local/nginx/sbin/nginxExecReload=/usr/local/nginx/sbin/nginx -s reloadExecStop=/usr/local/nginx/sbin/nginx -s quitPrivateTmp=true[Install]WantedBy=multi-user.target

nginx服务授权 && 重载服务服务​

sudo chmod 755 /usr/lib/systemd/system/nginx.service && sudo systemctl daemon-reload

开放80端口防火墙 && 重载防火墙服务​

sudo firewall-cmd --zone=public --add-port=80/tcp --permanent && sudo firewall-cmd --reload

nginx服务命令​

systemctl start nginx       # 开启nginx
systemctl status nginx      # 关闭nginx
systemctl enable nginx      # 自启动ninx
systemctl restart nginx     # 重启nginx
